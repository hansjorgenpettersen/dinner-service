# UI Styling Design

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add Tailwind CSS + shadcn/ui to the existing React SPA with a warm, food-focused visual design.

**Architecture:** Install Tailwind CSS v3 and shadcn/ui into the existing Vite/React frontend using the `shadcn` CLI (which handles tsconfig paths, CSS variables, and the `cn` utility automatically). **Tech Stack:** Tailwind CSS v3 (`tailwindcss@^3.4`), shadcn/ui CLI, postcss, autoprefixer, Vite

## Setup Steps

### 1. Install Tailwind CSS v3

```bash
cd frontend
npm install -D tailwindcss@^3.4 postcss autoprefixer
npx tailwindcss init -p  # generates tailwind.config.js + postcss.config.js
```

Tailwind v3 integrates with Vite via PostCSS â€” no changes to `vite.config.ts` needed. The `postcss.config.js` generated by `init -p` is sufficient.

### 2. Initialise shadcn/ui

```bash
npx shadcn@latest init
```

The CLI will:
- Add `"paths": { "@/*": ["./src/*"] }` and `"baseUrl": "."` to `tsconfig.json`
- Add the corresponding `resolve.alias` to `vite.config.ts`
- Create `src/lib/utils.ts` with the `cn()` helper
- Write CSS variable declarations into `src/index.css`

After init, replace the generated `src/index.css` with the warm palette skeleton (see below).

### 3. Add shadcn components

```bash
npx shadcn@latest add button input label card badge
```

This installs the five components plus their Radix UI peer dependencies:
- `@radix-ui/react-slot` (Button)
- `@radix-ui/react-label` (Label)
- `class-variance-authority`, `clsx`, `tailwind-merge` (all components)

### 4. Install lucide-react (icons)

```bash
npm install lucide-react
```

## tailwind.config.js

Configure only the `content` paths â€” **do not** add custom color names to `extend.colors`. All color values are applied as inline hex (e.g. `bg-[#7a3a1a]`) directly in JSX, keeping the config minimal:

```js
/** @type {import('tailwindcss').Config} */
export default {
  content: ['./index.html', './src/**/*.{ts,tsx}'],
  theme: { extend: {} },
  plugins: [],
}
```

## src/index.css

Replace the entire file with this skeleton after `shadcn init`. CSS variable overrides go inside `@layer base` to respect Tailwind's cascade:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer base {
  :root {
    /* shadcn requires HSL channel triplets (H S% L%) â€” no hsl() wrapper.
       Components reference these as hsl(var(--primary)) etc. */
    --background: 33 80% 96%;        /* #fdf6ee */
    --foreground: 26 77% 14%;        /* #3d1f08 */
    --card: 0 0% 100%;               /* #ffffff */
    --card-foreground: 26 77% 14%;   /* #3d1f08 */
    --border: 34 61% 77%;            /* #e8c9a0 */
    --primary: 24 65% 48%;           /* #c96a2b */
    --primary-foreground: 33 100% 97%; /* #fff8f0 */
    --muted-foreground: 32 36% 35%;  /* #7a5c3a */
    --destructive: 0 72% 51%;        /* #dc2626 */
    --destructive-foreground: 0 0% 100%;
    --radius: 0.5rem;
  }

  body {
    @apply bg-[#fdf6ee] text-[#3d1f08];
  }
}
```

> Note: shadcn CSS variables use **HSL channel triplets** (`H S% L%`), not hex or RGB. Components consume them as `hsl(var(--primary))`. The hex values in the Color Palette table are human-readable references; the HSL values above are what goes in `index.css`.

## Layout Structure

### Top Navbar (`Layout.tsx`)
- Full-width dark brown bar (`#7a3a1a`)
- Left: app name "đźŤ˝ Dinner Service" in cream

Every page follows the same structure:
```tsx

  <div className="flex items-center justify-between mb-6">

    <Button>+ Add New</Button>   {/* if applicable */}
  </div>
  {/* content */}
</main>
```

## Component Patterns

### List Items (Recipes, Shopping Lists)
White card with warm border, row layout:
```tsx
<div className="bg-white border border-[#e8c9a0] rounded-lg px-4 py-3 flex items-center gap-3">
  <span className="text-xl">{emoji}</span>
  <span className="font-medium text-[#3d1f08] flex-1">{name}</span>
  <ChevronRight className="text-[#c96a2b] w-4 h-4" />
</div>
```

### Forms (Login, Register, Add items)
- `Label` + `Input` stacked vertically, full width, gap-2 between pairs
- Primary `Button` for submit, full width on auth pages
- Muted text (`text-[#7a5c3a]`) for secondary actions / links

### Cards
```tsx
<Card className="border-[#e8c9a0] shadow-sm">
  <CardContent className="p-4">...</CardContent>
</Card>
```

### Buttons
- **Primary:** `<Button className="bg-[#c96a2b] hover:bg-[#a8571f] text-white">`
- **Outline:** `<Button variant="outline" className="border-[#e8c9a0] text-[#7a5c3a] hover:bg-[#fdf0e0]">`
- **Destructive:** `<Button variant="destructive">`

## shadcn/ui Components Used

| Component | Used in |
|---|---|
| `Button` | All pages |
| `Input` | Login, Register, ForgotPassword, ResetPassword, inline forms in detail pages |
| `Label` | All form pages |
| `Card`, `CardContent` | Dashboard stat cards, auth page wrapper |
| `Badge` | Shopping list category labels, shared-list indicator |

Note: `Dialog` is **out of scope**. Delete actions use inline confirmation (e.g. `window.confirm`) or destructive buttons without a modal.

## Page-by-Page Styling Summary

| Page | Key styling notes |
|---|---|
| Login / Register / Forgot / Reset | Centered `Card` on warm bg, `max-w-sm mx-auto`, logo/title at top, form inside card |
| Dashboard | 2-column grid of stat `Card`s (recipes count, shopping lists count) with links |
| Recipes | List of row items (recipe name + chevron); "+ New Recipe" button top-right |
| Recipe Detail | Title + edit/delete buttons; ingredients as list rows; image thumbnail if present; action buttons at bottom |
| Shopping Lists | List rows with list name + `Badge` for owner/shared status + chevron |
| Shopping List Detail | Items grouped by category heading; checkbox + item name per row; strikethrough when checked; "+ Add Item" at bottom |
| Products | Two-column layout: left 1/3 = category list with "+ Add Category"; right 2/3 = products filtered by selected category with "+ Add Product". Add a `selectedCategoryId: number \| null` state; clicking a category row in the left column sets it and filters the products list on the right. |
| Uncategorized Products | Simple list rows; each row has product name + category `<select>` dropdown |
| User | Profile info `Card`; default shopping list `<select>`; "Leave" buttons for shared lists; logout `Button` at bottom |

## Out of Scope

- Dark mode toggle
- Animations / transitions beyond Tailwind defaults
- Mobile-specific bottom nav or hamburger menu
- Image-heavy recipe card grid (list layout chosen)
- Custom icon set (use lucide-react icons from shadcn install)
- i18n / localization
- Confirmation dialogs (Dialog component) â€” use inline destructive buttons
